## 1. 文档说明

### 1.1 背景

本文档面向从事 ISO 认证、质量管理体系（QMS）、环境管理体系（EMS）、职业健康安全（OH&S）等认证服务的公司，旨在描述一套覆盖市场开拓、客户维护、合同履行、任务执行全流程的 ERP 系统需求。

### 1.2 适用范围

- 销售/市场部门：线索管理、报价、合同签订
- 审核/技术部门：审核任务分配与执行
- 财务部门：合同付款、发票、收款跟踪
- 管理层：经营数据统计与报表

### 1.3 术语说明

| 术语 | 说明 |
|------|------|
| 线索 | 潜在客户的初步联系信息 |
| 商机 | 经过初步评估、具备成交可能的销售机会 |
| 认证项目 | 一次完整的认证服务交付，包含初审、正式审核、发证等阶段 |
| 审核员 | 执行现场或文件审核的技术人员 |
| 体系 | 认证标准类型，如 ISO 9001、ISO 14001 等 |

## 2. 系统整体架构

### 2.1 架构概览

**架构类型**：B/S（浏览器/服务器）架构，支持公网 Web 访问

**部署环境**：Mac Studio（本地服务器），通过公网域名/IP 对外提供服务

```
  用户浏览器（Chrome / Edge / Safari）
         │  HTTPS
         ▼
┌─────────────────────────────────────────────────────────┐
│               Mac Studio（部署服务器）                   │
│                                                         │
│  ┌───────────────────────────────────────────────────┐  │
│  │  反向代理（Nginx）                                 │  │
│  │  · HTTPS 终止 / SSL 证书                          │  │
│  │  · 静态资源分发                                    │  │
│  └───────────────────────┬───────────────────────────┘  │
│                          │                              │
│  ┌───────────────────────▼───────────────────────────┐  │
│  │  应用服务层                                        │  │
│  │  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────┐  │  │
│  │  │ 市场需求 │ │ 客户管理 │ │订单/合同 │ │ 任务 │  │  │
│  │  │   管理   │ │          │ │  /付款   │ │ 管理 │  │  │

│  └───────────────────────┬───────────────────────────┘  │
│                          │                              │
│  ┌───────────────────────▼───────────────────────────┐  │
│  │  数据层                                            │  │
│  │  关系型数据库（PostgreSQL / MySQL）                │  │
│  │  文件存储（本地磁盘，建议挂载外置 NAS 或扩展盘）  │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

**公网访问方式**：

| 方案 | 说明 | 当前状态 |
|------|------|----------|
| Cloudflare Tunnel | 无需固定 IP，通过 Cloudflare 网络反代，自带 CDN 和 DDoS 防护，SSL 自动管理，免费 | **当前采用（动态 IP 环境）** |
| 固定公网 IP + 域名 | 宽带运营商分配固定 IP，绑定域名，直连延迟更低 | 后续可切换 |

> 系统架构设计时两种方案均兼容，Nginx 反向代理层不变，切换时仅需调整网络接入配置。

### 2.2 业务流程总览

```
市场线索录入
    │
    ▼
商机跟进 → 报价 → 合同签订
                      │
                      ▼
              客户档案建立/更新
                      │
                      ▼
              任务分配（审核计划）
                      │
                      ▼
              任务执行（现场/文件审核）
                      │
                      ▼
              审核报告 → 发证
                      │
                      ▼
              付款跟踪 → 发票 → 收款确认
```

### 2.3 模块关系

| 来源模块 | 触发事件 | 目标模块 |
|----------|----------|----------|
| 市场需求管理 | 商机转化成功 | 客户管理（新建/更新客户） |
| 客户管理 | 签订合同 | 订单/合同/付款管理 |
| 订单/合同管理 | 合同生效 | 任务管理（生成审核任务） |
| 任务管理 | 审核完成 | 订单管理（更新项目状态） |
| 订单管理 | 到期付款 | 付款管理（付款提醒） |

## 3. 技术选型

### 3.1 技术栈总览

| 层级 | 选型 | 说明 |
|------|------|------|
| 前端框架 | Vue 3 + Vite | 现代响应式框架，开发体验好，打包性能高 |
| 前端 UI 组件库 | Element Plus | 后台管理系统首选，组件完整，与 Vue 3 原生适配 |
| 后端框架 | FastAPI（Python） | 轻量高性能，异步支持，自动生成 OpenAPI 文档 |
| 数据库 | PostgreSQL | 功能完善，JSON 支持好，适合复杂业务查询 |
| ORM | SQLAlchemy 2.x + Alembic | 成熟的 Python ORM，Alembic 管理数据库迁移 |
| 认证授权 | JWT（JSON Web Token） | 无状态认证，适合前后端分离架构 |
| 反向代理 | Nginx | HTTPS 终止、静态资源分发、请求转发 |
| 容器化 | Docker + Docker Compose | 多服务编排，方便部署、升级和备份 |
| 公网接入 | Cloudflare Tunnel（当前） | 动态 IP 环境下的公网访问，后续可切换固定 IP |
| 文件存储 | 本地磁盘（Nginx 静态服务） | 合同、发票、证书等附件存储于 Mac Studio 本地 |

### 3.3 API 设计规范

- RESTful 风格，路径前缀 `/api/v1/`
- 统一响应格式：`{ "code": 0, "data": ..., "message": "ok" }`
- 认证方式：Bearer Token（JWT），Header：`Authorization: Bearer <token>`
- 分页参数统一：`?page=1&page_size=20`
- 文件上传：`multipart/form-data`，返回文件访问 URL

## 5. 模块一：市场需求管理

### 5.1 功能概述

管理从外部线索获取到商机确认转化的全过程，支持销售漏斗分析，帮助销售团队系统化跟进潜在客户。

### 5.2 核心功能点

#### 4.2.1 线索管理

- 手动录入线索（来源：电话、展会、网站询盘、转介绍等）
- 批量导入线索（Excel 模板）
- 线索自动去重（按公司名称 + 联系电话）
- 线索分配给指定销售人员
- 线索状态流转：`待跟进 → 跟进中 → 已转化 / 已放弃`
- 跟进记录（时间、方式、内容、下次跟进计划）

#### 4.2.2 商机管理

- 从线索一键创建商机，或直接新建商机
- 商机阶段管理：`初步接触 → 需求确认 → 报价 → 谈判 → 赢单 / 输单`
- 商机关联认证项目类型（ISO 9001 / ISO 14001 / ISO 45001 等）
- 预计金额与预计成交日期
- 竞争对手信息记录
- 失单原因记录（用于后续分析）
- 商机转化为客户 + 合同（赢单后一键操作）

#### 4.2.3 报价管理

- 基于服务项目生成报价单
- 支持多版本报价对比
- 报价单 PDF 导出/发送
- 报价有效期管理
- 折扣申请（触发审批流）

#### 4.2.4 市场分析

- 销售漏斗图（按阶段统计商机数量与金额）
- 线索来源分析（按渠道统计转化率）
- 销售人员业绩看板
- 月度/季度/年度商机转化报表

## 6. 模块二：客户管理

### 6.1 功能概述

集中管理认证服务客户的基本信息、认证历史、联系人及往来记录，为合同签订和任务执行提供完整的客户数据支撑。

### 6.2 核心功能点

#### 6.2.1 客户档案

- 客户基本信息维护（公司信息、营业执照、法人信息）
- 客户分类管理（按行业、规模、区域、重要级别）
- 客户来源记录（关联商机转化历史）
- 客户状态管理：`潜在客户 → 在服务 → 已到期 → 已流失`
- 多联系人管理（支持设置主要联系人）
- 附件管理（营业执照、资质证书扫描件等）
- 客户合并（处理重复录入）

#### 6.2.2 认证信息管理

- 每个客户可关联多个认证项目
- 证书信息管理：证书编号、认证范围、颁发日期、到期日期
- 证书状态：`有效 / 暂停 / 撤销 / 已过期`
- 证书到期自动预警（提前 3 个月、1 个月通知）
- 证书扫描件上传与归档

#### 6.2.3 客户跟进记录

- 销售/客服记录客户沟通情况（来电、拜访、邮件等）
- 跟进记录时间线展示
- 设置跟进提醒

#### 6.2.4 客户统计与分析

- 客户价值分析（历史合同总金额、合同数量）
- 到期续签预警列表
- 客户流失分析

## 7. 模块三：订单/合同/付款管理

### 7.1 功能概述

管理认证服务合同从起草、审批、签订到履行、付款、开票的完整生命周期，确保财务数据准确，到期款项不遗漏。

### 7.2 核心功能点

#### 7.2.1 合同管理

- 从商机/客户一键创建合同
- 合同编号自动生成（规则可配置，如：HT-2026-0001）
- 合同模板管理（按认证类型存储标准合同模板）
- 合同起草与编辑（富文本编辑器）
- 合同审批流（金额 ≥ 5万需管理层审批）
- 合同状态流转：`草稿 → 审批中 → 待签署 → 执行中 → 已完成 → 已终止`
- 合同附件上传（扫描件、电子签署文件）
- 合同变更记录（变更原因、变更内容、变更日期）

#### 7.2.2 服务项目管理

- 合同内服务项目明细（名称、认证标准、人天数、单价、金额）
- 多次认证周期管理（初审、监督审核、再认证）
- 关联审核任务（合同生效后自动生成任务计划）

#### 7.2.3 付款计划管理

- 合同签订时设置付款计划（支持多期付款，如首付 + 尾款）
- 付款节点与金额录入
- 到期自动提醒（提前 7 天、3 天）
- 付款记录：实收日期、实收金额、付款方式、备注

#### 7.2.4 发票管理

- 根据付款记录申请开票
- 发票信息：发票类型（增值税普票/专票）、发票抬头、税号、金额
- 发票状态：`待开具 → 已开具 → 已邮寄 / 已上传`
- 发票文件上传存档

#### 7.2.5 财务统计

- 应收款汇总（按客户、合同、时间段）
- 实收款与未收款对比
- 合同到期/续签预警
- 月度收款统计报表
- 发票台账导出

## 8. 模块四：任务管理

### 8.1 功能概述

管理认证审核过程中的所有任务，包括审核计划制定、审核员分配、任务执行过程记录，以及审核报告的提交与归档。

### 8.2 核心功能点

#### 8.2.1 认证项目管理

- 合同生效后自动创建认证项目
- 认证项目关联合同、客户、认证标准
- 项目阶段管理：`待启动 → 文件审查 → 审核准备 → 现场审核 → 报告整理 → 发证 → 监督/再认证`
- 项目进度追踪（整体完成百分比）
- 项目相关文档集中存储

#### 8.2.2 任务分配

- 审核主管基于项目创建审核任务
- 任务类型：文件审查任务 / 现场审核任务 / 监督审核任务 / 再认证任务 / 报告编写任务
- 分配审核员（支持多人协同）
- 设置任务开始日期、结束日期、优先级
- 审核员排期冲突检查（同一时间段已有任务则提示）
- 任务通知（分配后自动发送站内信 / 邮件给审核员）

#### 8.2.3 任务执行

- 审核员查看个人任务列表（日历视图 / 列表视图）
- 任务状态更新：`待开始 → 进行中 → 待审阅 → 已完成 / 已取消`
- 审核记录填写（不符合项、观察项、审核发现）
- 任务附件上传（审核记录表、证据材料、签到表）
- 工时记录（实际工时，用于成本核算）
- 任务完成后提交审核主管审阅

#### 8.2.4 审核报告管理

- 审核员在线填写或上传审核报告
- 审核主管审阅并批准报告
- 报告版本管理
- 报告发送给客户（邮件/下载链接）
- 不符合项跟踪（客户整改回复、关闭验证）

#### 8.2.5 任务统计

- 审核员工作量统计（任务数、工时）
- 任务完成率统计
- 逾期任务预警
- 认证项目进度总览

## 10. 非功能性需求

### 10.1 性能要求

| 指标 | 要求 |
|------|------|
| 页面响应时间 | 普通查询页面 ≤ 2 秒 |
| 报表生成时间 | 复杂报表 ≤ 10 秒 |
| 并发用户数 | 支持 ≥ 50 人同时在线 |
| 数据存储 | 支持 5 年以上历史数据查询 |

### 10.2 部署要求

| 项目 | 说明 |
|------|------|
| 服务器 | Mac Studio（本地部署） |
| 反向代理 | Nginx（HTTPS 终止、静态资源、请求转发） |
| SSL 证书 | Let's Encrypt 免费证书或商业证书，强制 HTTPS |
| 公网访问 | 当前使用 Cloudflare Tunnel，后续可切换固定 IP + 域名（见 2.1） |
| 数据备份 | 本地每日自动备份，建议同时备份至外部存储或云盘 |
| 防火墙 | 仅开放 443（HTTPS）和 22（SSH 管理）端口 |

### 10.3 安全要求

- 强制 HTTPS，禁止 HTTP 明文访问
- 用户密码加密存储（bcrypt 或同等强度）
- 操作日志完整记录（用户、时间、操作内容、IP）
- 数据权限隔离（销售人员默认只见自己负责的客户）
- 合同、发票等敏感文件访问需鉴权
- 登录失败锁定机制（防暴力破解）
- 每日数据备份

### 10.4 可用性要求

- 系统可用性尽力保障（单机部署，Mac Studio 重启会导致短暂中断）
- 支持主流浏览器（Chrome、Edge、Safari）
- 移动端支持（响应式设计，适配手机浏览器）

### 10.5 数据导出

- 所有列表页面支持导出为 Excel
- 合同、报价单支持导出为 PDF
- 财务报表支持导出为 Excel / PDF
